## Overview

JadeERP is a microservices-based Enterprise Resource Planning (ERP) application designed for building material and bulk commodity distribution businesses, including stone, sand, cement, gravel, and other construction materials.

The project focuses on modeling real-world enterprise workflows by separating each business capability into independent services. Instead of building a simple CRUD application, JadeERP aims to demonstrate scalable backend architecture using Domain-Driven Design (DDD), gRPC communication, and event-driven architecture.

The idea for JadeERP originated from casually watching videos about construction material distributors and quarry businesses. Exploring how these companies manage procurement, inventory, warehouse operations, deliveries, invoicing, and accounting inspired the development of a realistic ERP platform centered around those business processes.

## Planned Features

- **Authentication Service**: Authentication, authorization, role-based access control (RBAC), permissions, and branch management.
- **Master Data Service**: Centralized management of products, suppliers, customers, categories, units, and price lists.
- **Procurement Service**: Handles purchase requests, purchase orders, supplier transactions, and goods receiving.
- **Inventory Service**: Responsible for stock management, stock reservations, stock mutations, stock adjustments, and stock opname.
- **Sales Service**: Manages quotations, sales orders, and customer sales workflows.
- **Warehouse & Delivery Service**: Handles picking, packing, shipment preparation, delivery orders, and shipment tracking.
- **Accounting Service**: Manages invoices, customer payments, accounts receivable, and financial records.
- **Notification Service**: Sends email and WhatsApp notifications based on business events.
- **Reporting Service**: Provides dashboards, operational reports, inventory analytics, sales reports, and financial summaries.

## Technologies (Planned)

- **Go**: Primary programming language for all backend services.
- **Gin**: API Gateway and external REST API.
- **gRPC**: Synchronous communication between microservices.
- **RabbitMQ**: Event-driven communication between services.
- **PostgreSQL**: Database per service architecture.
- **Redis**: Distributed caching and temporary data storage.
- **Docker**: Containerization for every service.
- **JWT Authentication**: Secure authentication and authorization.

## Challenges and Learning Objectives

One of the primary goals of JadeERP is learning how to design and implement a production-inspired distributed system rather than focusing solely on application features. The project explores service boundaries, inter-service communication, database isolation, asynchronous event processing, and scalable deployment strategies.

Another challenge is modeling realistic ERP workflows while keeping each microservice independent. Business operations such as procurement, inventory management, warehouse processing, accounting, and reporting must communicate efficiently without becoming tightly coupled.

## Expected Outcome

JadeERP is intended to become a comprehensive portfolio project demonstrating enterprise software engineering practices and modern backend architecture.

Upon completion, the project will showcase:

- Domain-Driven Design (DDD)
- Microservices Architecture
- gRPC-based Service Communication
- Event-Driven Architecture using RabbitMQ
- Database per Service Pattern
- Containerized Deployment with Docker
- Enterprise-grade Backend Development using Go
