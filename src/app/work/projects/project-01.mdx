## Overview

Developed a web-based information system using Nuxt (Vue.js + TypeScript) for the frontend and Laravel 12 (PHP) for the backend. The application was designed with a scalable architecture, secure authentication, and integrated online payment processing through Midtrans. System design and planning were documented using UML diagrams to ensure a well-structured development process.

## Key Features

- **Modern Web Application**: Developed a responsive and user-friendly web application using Nuxt, Vue.js, and TypeScript, providing an intuitive interface across desktop and mobile devices.
- **Backend API Development**: Built RESTful APIs with Laravel 12 (PHP) to manage business logic, user management, transactions, and data processing.
- **Payment Gateway Integration**: Integrated the Midtrans Payment Gateway to support multiple payment methods, including QRIS, Virtual Account, and other supported payment channels, enabling secure and reliable online transactions.
- **Database Management**: Designed and managed a PostgreSQL database with optimized relational structures to ensure efficient data storage and retrieval.
- **System Design: Created UML diagrams**, including Use Case Diagrams, Activity Diagrams, Sequence Diagrams, and Entity Relationship Diagrams (ERD), to model system requirements and architecture before implementation.
- **Version Control and Collaboration**: Utilized Git and GitHub for version control, collaborative development, code reviews, and maintaining a structured development workflow.

## Technologies Used

- **Figma**: For creating and managing design assets and prototypes.
- **Laravel 12**: For building the backend of the application.
- **Nuxt, Vue.js, TypeScript**: For building the frontend of the application.
- **PostgreSQL**: For the database.
- **Midtrans Payment Gateway**: For payment processing.
- **UML Diagrams**: For system design and planning.
- **Git and GitHub**: For version control and collaboration.

## Challenges and Learnings

One key challenge was integrating the Midtrans Payment Gateway, which required understanding their documentation and implementing secure payment processing. Additionally, designing and managing a PostgreSQL database with optimized relational structures to ensure efficient data storage and retrieval was a learning experience that greatly improved the process.

## Outcome

- Know how to integrate third party app, read the documentation and implement it.
- Know how to design and manage a database with optimized relational structures to ensure efficient data storage and retrieval.
